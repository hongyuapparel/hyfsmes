# 订单成本页设计

> 本文档描述「订单成本」页面的信息架构、数据来源、计算逻辑与交互，便于跟单报价时快速算成本并得出工厂出厂价。

---

## 一、业务目标与公式

### 1.1 成本构成

订单成本由三部分汇总：

| 组成部分   | 数据来源                     | 说明 |
|------------|------------------------------|------|
| **物料信息** | 订单编辑 C 区，自动同步到本页，也可以手动编辑添加 | 面料/辅料等，每行需有「单价」才能参与成本汇总 |
| **工艺项目** | 订单编辑 E 区，自动同步到本页，也可以手动编辑添加 | 工艺项目（如印花、绣花等），每行需有「单价」才能参与成本汇总 |
| **生产工序** | 后台配置（每个工序一个单价）   | 跟单报价时在本页勾选/选择本单涉及的工序，系统按配置价自动汇总 |

### 1.2 计算公式

- **总成本** = 物料小计 + 工艺项目小计 + 生产工序小计  
- **工厂出厂价** = 总成本 ÷ (1 - 利润率)

说明：利润率以小数形式输入

---

## 二、页面结构建议

### 2.1 整体布局（自上而下）

1. **顶部：订单摘要（只读）**  
   订单号、SKU、客户、数量、当前销售价等，方便确认是哪一个单在做成本。

2. **第一块：物料成本**  
   - 表格列：物料类型、供应商、物料名称/颜色、单件用量、损耗%、订单件数、采购总量、**单价（元）**、**金额（元）**、备注。  
   - 数据来自订单详情接口的 `materials`，与订单编辑一致；**单价**在本页填写或从后续扩展的「物料参考价」带出，**金额 = 采购总量 × 单价**（或按你们实际规则，如按单件用量×订单数量×单价等，可在此固定一种规则）。  
   - 底部一行：**物料小计**。

3. **第二块：工艺项目成本**  
   - 表格列：工艺项目、供应商、**单价（元）**、**金额（元）**、工艺说明/备注。  
   - 数据来自订单详情的 `processItems`；**单价**在本页填写，**金额 = 单价 × 数量**（数量若按订单件数统一，可固定为订单数量或单独列「数量」列）。  
   - 底部一行：**工艺项目小计**。

4. **第三块：生产工序成本**  
   - 从后台配置的「生产工序列表」拉取（工序名称 + 单价）。  
   - 展示为可多选的列表（如复选框或可多选下拉），跟单勾选本单涉及的工序即可；**工序小计 = 所选工序单价之和**（若将来支持按件计费再扩展）。  
   - 底部一行：**生产工序小计**。

5. **第四块：成本汇总与出厂价**  
   - 三行展示：物料小计、工艺项目小计、生产工序小计。  
   - 一行：**总成本** = 上述三项之和。  
   - 一行：**利润率**（输入框，如 15 表示 15%，或 0.15 表示 15% 由产品统一约定）。  
   - 一行：**计算得出出厂价** = 总成本 × (1 + 利润率)。  
   - 可选：提供「按成本÷(1-利润率)」的第二种计算方式并切换显示，满足不同报价习惯。

6. **底部操作**  
   - 「应用到订单」：将计算得出的出厂价写回订单的 `ex_factory_price`，并可选提示「是否同时更新销售价」。  
   - 「仅计算」：不写回，仅在本页查看结果。

### 2.2 与订单编辑的同步关系

- **物料信息、工艺项目**：仅从订单编辑同步到成本页展示与参与计算，不在成本页增删行；若需要在成本页看到最新物料/工艺，由用户从订单编辑保存后再打开成本页（或成本页打开时调订单详情接口拉取最新数据）。  
- 成本页上为物料/工艺填写的**单价、金额**是否回写到订单：建议**不写回**订单编辑的 C/E 区原始结构，而是作为「成本快照」单独存（见下文数据设计），避免与订单编辑的纯业务字段混淆；若产品希望单价也同步回编辑页，再单独约定字段与接口。

## 三、后端数据与接口

### 3.1 生产工序配置（新增）

- **表**：如 `production_processes`（生产工序表）。  
- **字段建议**：`id`、`name`（工序名称）、`unit_price`（单价，decimal）、`sort_order`（排序）、`created_at`、`updated_at`。  
- **用途**：系统设置或「工艺/工序管理」中维护工序名称与报价；订单成本页通过接口拉取列表供勾选。

- **接口建议**：  
  - `GET /production-processes` 或 `GET /settings/production-processes`：列表（名称、单价），用于成本页下拉/多选。  
  - 增删改：在系统设置或独立「工序报价配置」页完成（与现有 process-items 工艺项目区分开：process-items 是「工艺项目名称」选项，不存单价；生产工序是带单价的工序库）。

### 3.2 订单成本快照（可选）

若希望保存每次计算结果、或把「单价」留在成本侧而不污染订单编辑：

- **表**：如 `order_cost_snapshots`。  
- **字段建议**：`order_id`、`material_total`、`process_item_total`、`production_process_total`、`total_cost`、`profit_margin`、`ex_factory_price`、`snapshot_at`（或仅保存最新一条，由业务决定）。  
- **接口**：  
  - 保存：`POST /orders/:id/cost-snapshot`（或 PUT 覆盖最新）；  
  - 读取：`GET /orders/:id/cost-snapshot`（成本页打开时带出上次单价与结果，便于微调）。

若首版不做快照，成本页每次打开仅拉订单详情 + 工序列表，单价全部在本页填写、不持久化也可。

### 3.3 订单详情接口

- 成本页打开时调用现有 **订单详情** 接口（与订单编辑同源），获取 `materials`、`processItems`、`quantity`、`exFactoryPrice` 等，用于展示与计算。

## 四、前端入口与路由

- **入口**：订单列表卡片底部「订单成本」图标（当前已有入口，点击后跳转或弹窗）。  
- **路由建议**：`/orders/cost/:id` 独立页面，或订单列表内弹窗（Dialog）内嵌成本表单。独立页便于大表格与多块内容展示，弹窗便于快速查看后关闭，可先选一种实现。

---

## 五、交互与体验要点

1. **只读与可编辑区分**：订单摘要、物料/工艺的「业务字段」（类型、名称、用量等）只读；单价、金额、利润率、工序勾选为可编辑。  
2. **自动汇总**：单价或勾选变化时，实时重算小计与总成本、出厂价，无需点「计算」按钮（可保留按钮作为「确认计算」的辅助）。  
3. **单位统一**：全页统一「元」或「元/件」，在表头或汇总区注明。  
4. **数量口径**：工艺项目、生产工序若按「单件」计费，需明确数量取订单数量还是其它；表格可增加「数量」列由用户填写或默认订单数量。  
5. **工序选择**：生产工序列表支持搜索、分页（若工序较多），勾选后即时参与汇总。

## 六、实施顺序建议

| 步骤 | 内容 |
|------|------|
| 1 | 后端：新增生产工序实体与表、列表接口；系统设置或独立页维护工序名称+单价 |
| 2 | 前端：订单成本页骨架（路由/入口）、订单摘要 + 物料/工艺只读展示，物料/工艺表格增加「单价」「金额」列并实现小计 |
| 3 | 前端：生产工序多选区与工序小计、总成本与利润率、出厂价计算与展示 |
| 4 | 后端：可选成本快照表与接口；前端「应用到订单」写回出厂价 |
| 5 | 优化：成本快照回显、单位与数量口径说明、第二种报价公式切换等 |

---

## 七、与现有模块的关系

- **订单编辑**：物料（C 区）、工艺项目（E 区）为数据来源，不在成本页改结构，仅改单价/金额用于成本。  
- **process-items（工艺项目）**：仅提供「工艺项目名称」选项，不带价格；成本页的工艺项目行来自订单的 `processItems`，单价在成本页填。  
- **生产工序**：新建的工序库（名称+单价），与 process-items 独立，专门用于跟单报价时勾选并汇总工序成本。

若你希望「工艺项目」也像生产工序一样在后台配置单价并在成本页带出，可再扩展工艺项目为「名称+默认单价」的配置表，成本页优先带出默认单价、允许修改；当前设计先采用成本页手工填工艺单价的方案，便于先上线再迭代。
