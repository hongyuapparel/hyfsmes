## 本地开发常用操作说明

本项目包含 **前端（Vue3 + Vite）** 和 **后端（NestJS + MySQL）**，以及使用 **Git + GitHub** 做代码备份。

## 重要环境前提（已确认，后续默认按此执行）

- 旧系统数据库在宝塔服务器，库名为 `hyfsmes`。
- 新系统数据库在阿里云（RDS），项目上线与迁移以阿里云库为准。
- 后端脚本里的“当前库”指 `backend/.env` 中 `MYSQL_DATABASE` 指向的库（即新系统库）。
- 脚本提示“若旧库不是当前库”时，意思是：
  - 若旧系统数据不在 `MYSQL_DATABASE` 指向的同一个库里，需要加参数 `--source-database <旧库名>`；
  - 如果旧系统在另一台数据库服务器（例如只在宝塔），就不能直接跨服务器读，需先把旧库数据导入到阿里云实例中的一个库，再用 `--source-database` 指向该库。

以后开发时，遇到下面几个问题，就按本文件操作即可：

- 如何打开数据库（MySQL）
- 如何启动后端
- 如何启动前端
- 如何把代码推送到 GitHub

## 一、宝塔拉取git上线

确认git更新后在宝塔终端：
cd /www/wwwroot/erp.hyfsmes.com/hyfsmes
sh scripts/deploy-full.sh

这会自动做：拉取 main、构建 backend/frontend、重启 PM2、发布前端。

然后回车，它会提示你输入密码，你就输入在 .env 里配置的那个密码（例如 123456），回车即可进入 MySQL。

登录成功后，切换到 erp 这个库：
USE erp;

查看有哪些表确认一下：
SHOW TABLES;

## 一、本地数据库（MySQL）如何打开

你在 cmd 里进入：
cd "C:\Program Files\MySQL\MySQL Server 8.0\bin"

之后，登录数据库的那条指令就是：
mysql -u root -p

然后回车，它会提示你输入密码，你就输入在 .env 里配置的那个密码（例如 123456），回车即可进入 MySQL。

登录成功后，切换到 erp 这个库：
USE erp;

查看有哪些表确认一下：
SHOW TABLES;

## 二、如何启动后端（NestJS）

> 记住：在 `backend` 目录执行 `npm run start:dev`。

### 1. 安装依赖（第一次必做）

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\backend"
npm install
```

### 2. 开发模式启动后端

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\backend"
npm run start:dev
```

- 默认端口：`http://localhost:3000`（由 `.env` 中的 `PORT` 决定）。
- 修改后端代码会自动重新编译、热重载。

> 以后每天开发：打开一个终端窗口，进入 `backend`，执行 `npm run start:dev` 即可。

## 三、如何启动前端（Vue3 + Vite）

> 记住：在 `frontend` 目录执行 `npm run dev`。

### 1. 安装依赖（第一次必做）

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\frontend"
npm install
```

### 2. 开发模式启动前端

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\frontend"
npm run dev
```

终端会输出类似：

```text
  Local:   http://localhost:5173/
```

在浏览器中访问这个地址即可看到系统界面。

> 以后每天开发：再打开一个终端窗口，进入 `frontend`，执行 `npm run dev`。

## 四、如何把代码推送到 GitHub

项目已配置好远程仓库：

- 远程名：`origin`
- 地址：`https://github.com/hongyuapparel/hyfsmes.git`
- 分支：`main`

每次想备份代码（推送到 GitHub），按下面 3 步执行即可。

### 1. 进入项目根目录

```bash
cd "e:\1.Cursor-Project\6. hyfsmes"
```

### 2. 提交到本地 Git 仓库

git  add .
```

（3）提交本次修改（引号里写本次修改说明）

git commit -m "本次修改说明"
```

### 3. 把提交推送到 GitHub

git push
```

如果是第一次推送，Git 可能会要求登录 GitHub 账号，按提示在浏览器中完成即可。

> 以后备份流程固定为：  
> **修改代码 → `add` → `commit` → `push`**

## 五、常见问题小记

- **前端打开后白屏 / 接口报错 500**  
  - 检查后端是否已启动：`npm run start:dev` 是否在运行。  
  - 检查数据库是否已启动，`backend/.env` 的 MySQL 配置是否正确。

- **Git 提示没有变化，不会提交**  
  - 说明你上次提交以来没有修改文件，正常现象；只有改动后再执行 `add` / `commit` 才会有新提交。

- **忘记命令怎么办？**  
  - 打开本文件 `docs/本地开发说明.md`，按对应章节步骤操作即可。
