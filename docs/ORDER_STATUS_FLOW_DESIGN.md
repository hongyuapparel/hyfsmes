## 二、订单状态设计

### 2.1 状态列表

请把所有订单状态列出来，并说明含义（下面是示例，可直接改成你的实际状态）：

| 状态 code | 中文名称 | 说明（何时进入、用来干什么） |
|----------|----------|------------------------------|
| `draft` | 草稿 | 新建订单但未提交，允许自由修改和删除 |
| `pending_review` | 待审单 | 业务员「保存并提交」后等待审核 |
| `pending_pattern` | 待纸样 | 审单通过后，等待纸样部门处理 |
| `pending_purchase` | 待采购 | 纸样确认后，等待采购下单 |
| `pending_cutting` | 待裁床 | 物料齐备后，等待裁床排产 |
| `pending_sewing` | 待车缝 | 裁床完成后，等待车缝 |
| `pending_finishing` | 待尾部 | 车缝完成后，等待尾部/质检/包装 |
| `completed` | 订单完成 | 所有工序完成并确认 |
| `canceled`? | 已取消 / 作废 | （如有，请补充说明何时进入） |

> 若你们有自己的一套状态命名，比如「待客户确认」「待打样」「样板确认」等，请按实际罗列。

## 五、字段与数据规则

这里专门写跟「状态」相关的关键字段规则，避免前后端理解不一致。

- **下单时间（orderDate）**  
  - 定义：第一次从 `draft` 进入非草稿状态时的时间（当前实现为「保存并提交」时写入）。  
  - 展示规则：  
    - 草稿状态订单：列表卡片上**不显示**下单时间。  
    - 其他状态且存在 `orderDate`：显示「下单时间：YYYY-MM-DD HH:mm」。  

- **订单类型（orderType / label）**  
  - 存储字段：数据库 `orders.label`。  
  - 前端字段：编辑页使用 `orderType`，保存时同步到 `label`。  
  - 展示：在订单卡片上用标签形式展示，仅显示值，不显示字段名。  

- **合作方式（collaborationType）**  
  - 存储字段：数据库 `orders.collaboration_type`。  
  - 展示：订单卡片 SKU 行后面用 `· 合作方式` 的形式追加，仅有值时显示。  

- **工艺项目（processItem）**  
  - 仅当编辑页选择了工艺时，列表卡片才以标签形式展示，无字段名。  

如有其它想固化的规则（例如「某状态不能删除」「审核必须填写原因」），也可以写在这里。
