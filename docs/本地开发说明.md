## 本地开发常用操作说明

本项目包含 **前端（Vue3 + Vite）** 和 **后端（NestJS + MySQL）**，以及使用 **Git + GitHub** 做代码备份。

以后开发时，遇到下面几个问题，就按本文件操作即可：

- 如何打开数据库（MySQL）
- 如何启动后端
- 如何启动前端
- 如何把代码推送到 GitHub

---

## 一、数据库（MySQL）如何打开

后端使用 MySQL，数据库连接配置在 `backend/.env` 中，对应示例如 `backend/.env.example`：

- 主机：`MYSQL_HOST=localhost`
- 端口：`MYSQL_PORT=3306`
- 用户：`MYSQL_USER=root`
- 密码：`MYSQL_PASSWORD=123456`
- 数据库名：`MYSQL_DATABASE=erp`

### 1. 安装并启动 MySQL（只需做一次）

1. 确认电脑已安装 MySQL（或使用 XAMPP、WAMP 等集成环境）。
2. 在 Windows「服务」中找到 `MySQL`（或对应服务名），确认状态为**正在运行**。

### 2. 创建数据库（只需做一次）

用你习惯的工具连接 MySQL（命令行 / 可视化工具均可），执行：

```sql
CREATE DATABASE erp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. 配置后端环境变量

1. 打开终端，进入后端目录：

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\backend"
```

2. 如果还没有 `.env` 文件，可以从示例复制一份（存在就不用重复）：

```bash
copy .env.example .env
```

3. 根据自己本地的数据库配置，修改 `.env` 中的：

- `MYSQL_HOST`
- `MYSQL_PORT`
- `MYSQL_USER`
- `MYSQL_PASSWORD`
- `MYSQL_DATABASE`

保存即可。

## 二、如何启动后端（NestJS）

> 记住：在 `backend` 目录执行 `npm run start:dev`。

### 1. 安装依赖（第一次必做）

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\backend"
npm install
```

### 2. 开发模式启动后端

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\backend"
npm run start:dev
```

- 默认端口：`http://localhost:3000`（由 `.env` 中的 `PORT` 决定）。
- 修改后端代码会自动重新编译、热重载。

> 以后每天开发：打开一个终端窗口，进入 `backend`，执行 `npm run start:dev` 即可。

## 三、如何启动前端（Vue3 + Vite）

> 记住：在 `frontend` 目录执行 `npm run dev`。

### 1. 安装依赖（第一次必做）

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\frontend"

```

### 2. 开发模式启动前端

```bash
cd "e:\1.Cursor-Project\6. hyfsmes\frontend"

```


> 以后每天开发：再打开一个终端窗口，进入 `frontend`，执行 `npm run dev`。

## 四、如何把代码推送到 GitHub

项目已配置好远程仓库：

- 远程名：`origin`
- 地址：`https://github.com/hongyuapparel/hyfsmes.git`
- 分支：`main`

每次想备份代码（推送到 GitHub），按下面 3 步执行即可。

### 1. 进入项目根目录

```bash
cd "e:\1.Cursor-Project\6. hyfsmes"
```

### 2. 提交到本地 Git 仓库

（1）可选：查看当前修改

```bash
& "C:\Program Files\Git\bin\git.exe" status
```

（2）把所有修改加入暂存区

```bash
& "C:\Program Files\Git\bin\git.exe" add .
```

（3）提交本次修改（引号里写本次修改说明）

```bash
& "C:\Program Files\Git\bin\git.exe" commit -m "本次修改说明"
```

### 3. 把提交推送到 GitHub

```bash
& "C:\Program Files\Git\bin\git.exe" push
```

如果是第一次推送，Git 可能会要求登录 GitHub 账号，按提示在浏览器中完成即可。

> 以后备份流程固定为：  
> **修改代码 → `add` → `commit` → `push`**

## 五、常见问题小记

- **前端打开后白屏 / 接口报错 500**  
  - 检查后端是否已启动：`npm run start:dev` 是否在运行。  
  - 检查数据库是否已启动，`backend/.env` 的 MySQL 配置是否正确。

- **Git 提示没有变化，不会提交**  
  - 说明你上次提交以来没有修改文件，正常现象；只有改动后再执行 `add` / `commit` 才会有新提交。

- **忘记命令怎么办？**  
  - 打开本文件 `docs/本地开发说明.md`，按对应章节步骤操作即可。
