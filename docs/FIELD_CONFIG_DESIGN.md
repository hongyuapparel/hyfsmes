# 字段系统配置设计方案

> 面向非技术用户：说明如何实现「同一字段全系统一致」「可自行调整顺序/隐藏」「跨页面字段关联」

---

## 一、您期望的效果

1. **同一字段全系统一致**：例如「公司名称」在客户管理、订单、产品等页面，显示名称、含义都相同
2. **可自行配置**：在系统里调整字段顺序、隐藏/显示，无需改代码
3. **跨页面关联**：不同页面的字段可以互相引用（如订单里的「客户」关联客户表的「公司名称」）

---

## 二、当前系统已有能力

系统**已经具备**字段配置的基础架构：

| 能力 | 状态 | 说明 |
|------|------|------|
| 字段配置表（数据库） | ✅ 已有 | `field_definitions` 表，存储模块、字段 code、显示名、顺序、是否可见等 |
| 字段配置 API | ✅ 已有 | 按模块获取、更新顺序、更新显示/隐藏 |
| 产品管理 | ✅ 已接入 | 产品页有「列设置」按钮，可调顺序、勾选显示/隐藏 |
| 客户管理 | ❌ 未接入 | 仍使用代码中的固定配置 |

**产品管理**的「列设置」就是您期望的配置方式：在页面内点击按钮，弹窗中勾选显示、上移/下移调整顺序，修改后立即生效。

---

## 三、推荐方案：统一接入字段配置

### 3.1 设计原则

- **单一数据源**：每个字段用唯一 `code`（如 `companyName`），全系统共用
- **按模块区分**：客户、产品、订单等各自有 `module`，但相同 `code` 的字段含义一致
- **配置优先**：页面优先从数据库读取配置，无配置时回退到代码默认值

### 3.2 实现步骤

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1 | 客户模块接入配置 | 在 seed 中写入客户字段；客户页增加「列设置」；从 API 读取配置 |
| 2 | 统一字段 code | 跨模块共用同一 code（如 `companyName`、`salesperson`），保证语义一致 |
| 3 | 跨页面关联预留 | 在配置中增加「关联模块/字段」等扩展字段，供后续订单-客户等关联使用 |

### 3.3 跨页面字段关联（后续扩展）

当需要「订单的客户」关联「客户表的公司名称」时：

- 订单模块的 `customerId` 字段，配置 `optionsKey: 'customers'`，选项来自客户表
- 产品模块的 `companyName` 已配置为「客户(公司名称)」，通过 `customer` 关联取客户的公司名
- 未来可在配置表增加 `linkModule`、`linkField` 等字段，实现更灵活的关联

---

## 四、配置入口位置建议

| 方案 | 入口 | 适用场景 |
|------|------|----------|
| A. 各页面内 | 每页顶部「列设置」按钮（如产品页已有） | 只配置当前页字段，操作直观 |
| B. 系统设置统一入口 | 系统设置 → 字段配置 → 选择模块 | 集中管理所有模块，适合管理员 |

**建议**：先采用方案 A（与产品页一致），各业务页自带「列设置」；后续如需集中管理，再增加方案 B。

---

## 五、安全与回退

- **安全**：配置存在数据库，有权限控制；误操作可通过重新 seed 或管理界面恢复
- **回退**：若 API 无数据，页面自动使用代码中的默认配置，不影响正常使用

---

## 六、总结

| 您的需求 | 对应方案 |
|----------|----------|
| 同一字段全系统一致 | 统一使用 `code`，配置表按模块存储 |
| 自行调整顺序/隐藏 | 各页「列设置」弹窗，调用已有 API |
| 跨页面字段关联 | 通过 `optionsKey`、关联查询实现；后续可扩展 `linkModule` 等 |

**下一步**：将客户管理接入字段配置（与产品管理一致），即可在客户页使用「列设置」自行调整顺序和显示/隐藏。
