# 后端数据库配置说明

## 无法登录 / 无法连接后端

若前端提示「无法连接后端，请确认后端已启动」：

1. **先启动后端**：在 `backend` 目录执行 `npm run start:dev`
2. 若后端启动失败并报 **MySQL Access denied**，说明数据库连接配置不对，请按下面步骤检查。

## MySQL 连接失败：Access denied

说明当前 `.env` 里的账号密码与本机 MySQL 不一致。

### 操作步骤

1. **确认本机已安装并启动 MySQL**
   - 服务中查看是否有 MySQL 服务且为「正在运行」
   - 或命令行执行：`mysql -u root -p` 能登录

2. **修改 `backend\.env`**
   - 用记事本或 VS Code 打开项目中的 `backend\.env`
   - 修改以下项为你本机实际配置：
     - `MYSQL_HOST`：一般为 `localhost`
     - `MYSQL_PORT`：一般为 `3306`
     - `MYSQL_USER`：一般为 `root`
     - **`MYSQL_PASSWORD`**：改成你本机 MySQL root 的密码；若无密码则留空：`MYSQL_PASSWORD=`
     - `MYSQL_DATABASE`：库名，默认 `erp`（需在 MySQL 中先创建：`CREATE DATABASE erp;`）

3. **保存后重新启动后端**
   ```powershell
   cd "e:\1.Cursor-Project\6. hyfsmes\backend"
   npm run start:dev
   ```
   看到 `Backend running at http://localhost:3000` 且无报错后，再刷新前端登录页进行登录。

## 默认管理员账号（首次启动并连接数据库后由系统创建）

- 用户名：见 `.env` 中 `ADMIN_USERNAME`（默认 `admin`）
- 密码：见 `.env` 中 `ADMIN_PASSWORD`（默认 `admin123`）

建议首次登录成功后到后台修改密码。
