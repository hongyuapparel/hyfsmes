# 鸿宇服饰 - 视觉设计系统指南

## 概述

这是一个为服装生产企业设计的现代化管理系统。设计采用**深色侧边栏 + 白色主内容区**的经典企业级布局，配合精心选择的色彩系统，确保专业性和易用性的完美平衡。

---

## 色彩系统

### 色彩哲学

整个设计系统基于**可信度 + 现代感**的原则：
- **深色侧边栏**（Slate-900）：传达稳定、专业、可信的品牌形象
- **白色内容区**：提供清晰的信息展示空间，降低认知负荷
- **蓝紫色系主色**：在科技感和专业感之间达到平衡
- **明黄色强调**：用于关键数据和行动点，提高视觉吸引力

### 核心色彩

| 色彩名称 | OKLch值 | 十六进制 | 用途 | 说明 |
|---------|---------|---------|------|------|
| **主色（Primary）** | oklch(0.52 0.18 265) | #5B6FD8 | 按钮、链接、强调 | 深蓝紫，具有高度识别性 |
| **次色（Secondary）** | oklch(0.58 0.16 200) | #6AAFD8 | 图表、渐变 | 冷调青蓝，与主色形成冷色系 |
| **强调色（Accent）** | oklch(0.65 0.15 70) | #D4C03F | 数据标签、重要指示 | 明黄色，用于突出关键信息 |
| **背景色（Background）** | oklch(0.98 0.001 0) | #F9F9FB | 页面背景 | 极浅灰，视觉舒适，不刺眼 |
| **卡片色（Card）** | oklch(1 0 0) | #FFFFFF | 内容容器 | 纯白，确保内容清晰 |
| **文字色（Foreground）** | oklch(0.25 0.02 250) | #3D4452 | 主要文本 | 深灰，与白色背景有足够对比 |
| **辅文字（Muted）** | oklch(0.55 0.01 250) | #7D8694 | 次要信息 | 中灰，用于不那么重要的内容 |
| **边框色（Border）** | oklch(0.92 0 0) | #EBEBF0 | 分割线、边框 | 极浅灰，建立视觉分层 |
| **侧边栏背景** | oklch(0.13 0.01 250) | #1F2937 | 导航背景 | 深蓝灰，对比度高 |
| **侧边栏文字** | oklch(0.95 0.01 250) | #F1F1F3 | 导航文字 | 浅白灰 |

### 色彩优化建议

**当前配色的优势：**
✅ 冷色系一致性强，显得专业高端
✅ 对比度充分，易于长时间阅读
✅ 色彩应用清晰，不显得混乱

**可优化方向：**
1. **强调色使用频率**：明黄色(Accent)目前未被充分利用，建议在重要数据、关键按钮上加强使用
2. **侧边栏活跃状态**：当前使用Slate-700灰色背景，可考虑加入轻微的蓝色叠加(bg-slate-700/80 + blue tint)来增强品牌感
3. **浮动元素**：建议对悬停态增加subtle shadow，增强交互反馈

## 布局系统

### 整体框架

```
┌─────────────────────────────────────────┐
│                顶部导航栏（h-16）        │
├──────────────┬─────────────────────────┤
│   侧边栏     │                         │
│  (w-56或16)  │    主内容区             │
│              │  (flex-1 overflow-auto) │
│              │                         │
│              │                         │
├──────────────┴─────────────────────────┤
```

### 尺寸标准

| 组件 | 尺寸 | 说明 |
|------|------|------|
| 顶部导航栏 | h-16 (64px) | 包含标题、搜索框、用户头像 |
| 侧边栏展开 | w-56 (224px) | 显示完整文本标签 |
| 侧边栏折叠 | w-16 (64px) | 仅显示图标 |
| LOGO区域 | h-14 (56px) | 顶部标识 |

### 响应式设计

- **侧边栏**：支持展开/折叠动画，提供更多主内容空间
- **菜单项文字**：使用 `truncate` 防止溢出
- **内容区**：使用 `flex-1 overflow-auto`，自适应高度

## 组件设计

### 1. 侧边栏（Sidebar）

**设计原则：**
- 深色背景降低视觉疲劳
- 标志区域保持紧凑（h-14）
- 菜单项采用柔和交互反馈

**菜单项状态：**
```
未活跃（默认）: 文字 slate-400，背景透明
悬停态:        文字 slate-300，背景 slate-800/50
活跃态:        文字 white，背景 slate-700
```

**特点：**
- 活跃菜单项使用灰色背景而非蓝色，避免视觉过于厚重
- 图标 + 文字排列，使导航清晰明确
- 底部用户区域采用更小的头像和文字，保持平衡

### 2. 顶部导航栏（Top Bar）

**组成部分：**
- 左侧：汉堡菜单 + 页面标题
- 右侧：搜索框 + 用户头像

**设计特点：**
- 纯白背景，与主内容区协调
- 极细边框(border-slate-200)分隔顶部和内容
- 搜索框采用浅灰背景(bg-slate-50)，清晰但不突兀

### 3. 内容卡片（Content Card）

**样式标准：**
```css
背景: 纯白 (#FFFFFF)
边框: 1px solid #EBEBF0
圆角: 8px (rounded-xl)
阴影: subtle (根据深度调整)
间距: p-6 (24px)
```

**卡片分层：**
- **L1 容器**：列表、表格外框
- **L2 组件**：单个数据卡、输入框
- **L3 元素**：图标、小按钮

### 4. 数据展示

**KPI 卡片：**
- 大数字（text-2xl, font-bold）+ 上升/下降百分比
- 左上角标题，左下角数据
- 右侧小圆形图标（p-3, rounded-lg, bg-primary/10）

**表格：**
- 表头：bg-muted/50，text-xs font-bold
- 行：border-b border-border，hover:bg-muted/30
- 状态标签：彩色背景 + 对应文字颜色

**图表：**
- 使用5种预定义图表色
- 确保色彩盲友好（不仅依赖颜色区分）

## 交互设计

### 过渡动画

| 组件 | 持续时间 | 缓动 | 作用 |
|------|---------|------|------|
| 侧边栏折叠/展开 | 300ms | ease-in-out | 流畅的面板动画 |
| 按钮悬停 | 200ms | ease-in-out | 快速响应 |
| 菜单项切换 | 200ms | ease-in-out | 清晰的状态变化 |

### 状态反馈

**按钮：**
- 默认：primary 色
- 悬停：深化颜色（opacity-90）或添加阴影
- 按下：进一步深化
- 禁用：灰化(opacity-50)

**输入框：**
- 默认：bg-slate-50, border-slate-200
- 焦点：ring-2 ring-primary/50，边框不变
- 错误：ring-2 ring-destructive/50

**菜单项：**
- 未活跃：低对比度，hover 时提升
- 活跃：清晰的背景色，强调效果

| xl | 20px | 大间距 |

**常见组合：**
- 卡片内边距：p-6 (24px)

## 无障碍考虑

✅ **色彩对比度**：所有文字与背景的对比度≥ 4.5:1，符合 WCAG AA 标准
✅ **非色彩指示**：活跃菜单项不仅用颜色表示，还通过背景层级表达
✅ **焦点状态**：所有交互元素都有清晰的焦点环(ring)
✅ **icon + label**：菜单项同时显示图标和标签，提高可识别性

## 配色优化建议

### 短期优化（立即实施）

1. **强调色应用**
   - 重要数据标签使用 Accent 色（明黄）
   - CTA 按钮考虑使用 Accent 色作为补充
   - 成功提示使用绿色

2. **侧边栏微调**
   - 活跃菜单项背景改为 slate-700/90（增加透明度和蓝色叠加）
   - 增加 subtle border-left 指示器，不占空间

3. **数据可视化**
   - 表格行状态色彩：成功(绿) / 进行中(蓝) / 待处理(灰) / 错误(红)

### 中期优化（持续迭代）

1. **渐变应用**
   - LOGO 区域的主色 → 次色渐变已应用
   - 考虑在大型卡片背景应用微妙渐变(opacity < 0.05)

2. **深度阴影体系**
   - 创建 3 级阴影：轻(悬停卡片)、中(浮动元素)、重(模态框)

3. **暗色模式支持**
   - 未来可考虑添加暗色模式变体

## 设计决策记录

| 决策 | 原因 | 备选方案 |
|------|------|---------|
| 深色侧边栏 + 白色主区 | 经典专业组合，降低眼睛疲劳 | 全白、全深色 |
| 蓝紫主色 | 科技感 + 专业性平衡 | 纯蓝、纯紫 |
| 灰色活跃菜单 | 避免过度强调，保持整体沉静 | 蓝色、彩色 |
| 明黄强调色 | 突出关键数据，色彩盲友好 | 橙色、绿色 |
